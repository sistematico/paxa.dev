# Folha de atalhos do Neovim

> **Importante:** atalhos marcados como dependentes da configuração não são universais. Execute `:map`, `:nmap`, `:imap`, `:vmap` e `:Telescope keymaps` dentro do Neovim para consultar os mapeamentos ativos.

### Notação

| Símbolo | Descrição |
|---|---|
| `<C-x>` | Ctrl + x |
| `<A-x>` | Alt + x |
| `<S-x>` | Shift + x |
| `<CR>` | Enter |
| `<Esc>` | Escape |
| `<leader>` | Tecla líder, normalmente Espaço |

# 6. Arquivos, buffers e janelas

## Arquivos

| Atalho ou comando | Descrição |
|---|---|
| `:w` | Salvar arquivo |
| `:wa` | Salvar todos os arquivos |
| `:q` | Fechar janela |
| `:q!` | Fechar sem salvar |
| `:wq` or `ZZ` | Salvar e fechar |
| `:e path` | Abrir arquivo |
| `:e!` | Recarregar descartando alterações |

## Buffers

| Atalho ou comando | Descrição |
|---|---|

| `:bprevious` or `:bp` | Buffer anterior |
| `:buffer {number}` | Abrir buffer pelo número |
| `:bdelete` or `:bd` | Fechar buffer atual |
| `<C-^>` | Alternar entre os dois últimos buffers |

| `<C-w>k` | Focar janela acima |

# 9. Comments — comment.lua

O mapeamento exato depende de qual plugin de comentários o arquivo `comment.lua` carrega.

Mapeamentos comuns usados pelo **Comment.nvim**:

| Atalho | Descrição |
|---|---|
| `gcc` | Comentar/descomentar linha atual |
| `gc{motion}` | Comentar/descomentar usando movimento |
| `gbc` | Alternar comentário em bloco |
| `gb{motion}` | Comentário em bloco usando movimento |
| `gc` in Visual mode | Comentar/descomentar seleção |
| `gb` in Visual mode | Alternar comentário em bloco da seleção |

Exemplos:

| Comando | Descrição |
|---|---|
| `gcip` | Comentar parágrafo atual |
| `gc3j` | Comentar linha atual e as próximas 3 |

# 10. Telescope — telescope.lua

Os atalhos globais que **abrem o Telescope dependem da configuração**. Mapeamentos personalizados comuns:

| Mapeamento comum | Descrição |
|---|---|
| `<leader>ff` | Procurar arquivos |
| `<leader>fg` | Buscar texto no projeto |
| `<leader>fb` | Procurar buffers abertos |
| `<leader>fh` | Buscar na ajuda |
| `<leader>fr` | Arquivos recentes |
| `<leader>fk` | Buscar atalhos |
| `<leader>fd` | Buscar diagnósticos |
| `<leader>fs` | Buscar símbolos do documento |

Dentro do Telescope:

| Atalho | Descrição |
|---|---|
| `<C-n>` or `<Down>` | Próximo resultado |
| `<C-p>` or `<Up>` | Resultado anterior |
| `<CR>` | Abrir resultado selecionado |
| `<C-x>` | Abrir em divisão horizontal |
| `<C-v>` | Abrir em divisão vertical |
| `<C-t>` | Abrir em nova aba |
| `<C-u>` | Rolar pré-visualização para cima |
| `<C-d>` | Rolar pré-visualização para baixo |
| `<Tab>` | Marcar item e ir ao próximo |
| `<S-Tab>` | Marcar item e ir ao anterior |
| `<C-q>` | Enviar resultados ao quickfix |
| `<Esc>` or `<C-c>` | Fechar Telescope |

Comandos úteis:

```vim
:Telescope find_files
:Telescope live_grep
:Telescope buffers
:Telescope help_tags
:Telescope oldfiles
:Telescope keymaps
:Telescope diagnostics
```

# 15. Formatting — conform.lua

Os atalhos de formatação dependem completamente da configuração.

Atalho comum:

| Mapeamento comum | Descrição |
|---|---|
| `<leader>f` | Formatar arquivo ou seleção |

Comando Lua típico:

```lua
require("conform").format({ async = true, lsp_format = "fallback" })
```

Inspeção útil:

```vim
:ConformInfo
```

# 18. Treesitter — treesitter.lua

O Treesitter melhora principalmente a análise sintática e o destaque de código. Ele pode não adicionar atalhos, a menos que a seleção incremental esteja configurada.

Mapeamentos comuns de seleção incremental:

| Mapeamento comum | Descrição |
|---|---|
| `<C-Space>` | Iniciar ou expandir seleção sintática |
| `<BS>` | Diminuir seleção sintática |

Comandos úteis:

```vim
:Inspect
:InspectTree
:TSInstallInfo
:TSUpdate
```

# 19. Indentation — indent.lua

Plugins de guias de indentação normalmente adicionam apenas indicadores visuais e podem não possuir atalhos.

Native indentation shortcuts:

| Atalho | Descrição |
|---|---|
| `>>` | Aumentar recuo da linha |
| `<<` | Diminuir recuo da linha |
| `>` in Visual mode | Aumentar recuo da seleção |
| `<` in Visual mode | Diminuir recuo da seleção |
| `=` | Reindentar usando movimento |
| `gg=G` | Reindentar arquivo inteiro |

# 20. Plugins de interface

Estes plugins normalmente não adicionam atalhos importantes de edição:

- `alpha.lua` — painel inicial
- `lualine.lua` — barra de status
- `tokyo-night.lua` — tema de cores
- `indent.lua` — guias de indentação

Os atalhos do painel Alpha dependem totalmente da configuração em `alpha.lua`.

# 21. Descobrir seus atalhos exatos

Execute estes comandos dentro do Neovim:

```vim
:map
:nmap
:imap
:vmap
:xmap
:map <leader>
:verbose nmap <leader>e
:verbose nmap gd
:Telescope keymaps
```

Significado:

| Comando | Descrição |
|---|---|
| `:map` | Mostrar atalhos de vários modos |
| `:nmap` | Mostrar atalhos do modo Normal |
| `:imap` | Mostrar atalhos do modo Inserção |
| `:vmap` | Mostrar atalhos do modo Visual |
| `:verbose nmap ...` | Mostrar o atalho e onde foi definido |
| `:Telescope keymaps` | Pesquisar atalhos ativos interativamente |

Para verificar a tecla líder:

```vim
:lua print(vim.g.mapleader)
:lua print(vim.g.maplocalleader)
```

## Nota de precisão

Os atalhos nativos do Neovim são estáveis, mas mapeamentos de plugins como `<leader>ff`, `<leader>e`, `gd`, formatação e navegação entre buffers costumam ser personalizados nos arquivos Lua. Use `:verbose map` ou `:Telescope keymaps` para confirmar a configuração ativa.
