Como os Prompts do ZSH Funcionam

Tradicionalmente, os shells definem o prompt usando a variável de ambiente PS1. O ZSH suporta isso para compatibilidade, mas também fornece maneiras mais avançadas de definir prompts usando as variáveis PROMPT e RPROMPT.

A variável PROMPT define o lado esquerdo do prompt, enquanto RPROMPT define o lado direito. Essas variáveis podem conter uma mistura de texto e sequências de escape que o ZSH expande quando o prompt é exibido.

A maioria dos prompts ZSH usa uma combinação de sequências de escape e funções personalizadas que são executadas quando o prompt é exibido. Essas funções podem exibir informações dinâmicas como o branch Git atual, o ambiente virtual Python atual ou a versão do NodeJS atual.
Construindo um Prompt ZSH Personalizado

Ao longo dos anos usando ZSH, passei a apreciar prompts minimalistas que são rápidos e discretos. Queria construir um prompt que exibisse apenas as informações essenciais, como o diretório de trabalho atual, branch e status do git. Algo assim:
text

~/projects/my-project (branch) [status]

$

O ZSH usa sequências de escape para exibir informações dinâmicas no prompt. Por exemplo, a sequência de escape %n expande para o nome de usuário atual, %~ exibe o diretório de trabalho atual e %# expande para # se o usuário atual tiver privilégios de superusuário.

Para iterar e testar o prompt rapidamente, você pode definir a variável PROMPT na sessão atual do shell. Por exemplo, o prompt padrão do ZSH no macOS é "%n@%m %~%# ", que exibe o nome de usuário, nome do host, diretório de trabalho e um # se o usuário tiver privilégios de superusuário. Seria equivalente a definir a variável PROMPT para:

```zsh
PROMPT='%n@%m %~%# '
```

Usando isso como ponto de partida, decidi me livrar do nome de usuário e do host (já que sou o único usuário na minha máquina) para terminar com algo assim:

```zsh
PROMPT='%~ %# '
```

Indo para Múltiplas Linhas

Sou um grande fã de prompts de várias linhas, pois me permitem exibir caminhos longos e mais informações sem sobrecarregar o prompt.

Minha primeira tentativa foi inserir um caractere de nova linha (\n) na variável PROMPT, mas isso não funcionou. O ZSH não suporta sequências de escape na variável PROMPT, então usei o hook precmd para imprimir o CWD e exibir o prompt na linha seguinte.

O hook precmd é executado antes de cada prompt ser exibido e pode ser usado para executar comandos ou funções personalizadas. Usei print com -P dentro dele para interpretar sequências de expansão de prompt, assim como na variável PROMPT.

```zsh
precmd() {
  print -P "%~"
}

PROMPT='%# '
```

Funcionou perfeitamente; pude exibir o diretório de trabalho atual em uma nova linha.
Adicionando Informações do Git

O ZSH tem suporte integrado para exibir informações do Git no prompt. Você pode usar o módulo vcs_info para exibir o branch Git atual e o status no prompt.

Para habilitar o módulo vcs_info, você deve carregá-lo em sua sessão ZSH usando o comando autoload.

```zsh
autoload -Uz vcs_info
```

Uma vez carregado, você deve invocar a função vcs_info no hook precmd para atualizar as informações do Git antes de exibir cada prompt. A função então preenche a variável vcs_info_msg_0_ com as informações do Git, que você pode exibir no prompt.

```zsh
precmd() {
  vcs_info
  print -P "%~ ${vcs_info_msg_0_}"
}
```

Exceto que isso não funcionou. O ZSH não habilita a expansão de variáveis em prompts por padrão, então tive que habilitá-lo usando o comando setopt prompt_subst.

```zsh
setopt prompt_subst
```

Voilà! Agora eu tinha um prompt ZSH personalizado que exibia o diretório de trabalho atual e as informações do Git.
Personalizando as Informações do VCS

Você pode personalizar as informações do Git exibidas no prompt definindo a variável vcs_info_msg_0_ para uma string de formato personalizada. A string de formato pode conter sequências de escape que o ZSH expande quando o prompt é exibido.

Eu queria ter o branch atual e o status de staged/unstaged. Então, usando zstyle, mudei a string de formato para:

```zsh
zstyle ':vcs_info:*' enable git
zstyle ':vcs_info:git:*' formats '%b%f %m%u%c %a'
```

Esta string de formato exibe o branch atual (%b), as mudanças staged (%c) e unstaged (%u), qualquer ação em andamento como rebase (%a) e informações diversas (%m).

Eu queria exibir símbolos em vez de texto para o status do Git, então defini as variáveis stagedstr e unstagedstr como + e !.

```zsh
zstyle ':vcs_info:*' check-for-changes true
zstyle ':vcs_info:*' stagedstr ' +'
zstyle ':vcs_info:*' unstagedstr ' !'
```

Cores e Formatação

Consegui fazer o prompt funcionar, mas estava sem graça. Queria adicionar algumas cores e formatação para destacá-lo ✨.

O ZSH suporta sequências de escape para cores e formatação, que você pode usar para estilizar o prompt. A sequência de escape %F{cor} define a cor do primeiro plano, %K{cor} define a cor do plano de fundo e %B define texto em negrito.

Eu queria exibir o diretório de trabalho atual em negrito, o branch do git em azul e o status do git em verde e vermelho para mudanças staged e unstaged. Também queria exibir símbolos Powerline para o prompt.

Então fui para o Nerd Fonts, baixei a fonte "Fira Code" modificada e a defini como fonte no iTerm.

Em seguida, usei sequências de escape para definir as cores e formatação nos estilos PROMPT e vcs_info.

Aqui está o prompt final. O símbolo Powerline para o branch git \uE0A0 infelizmente não é suportado em nenhuma outra fonte e, portanto, aparece como um quadrado não reconhecido.

```zsh
zstyle ':vcs_info:git*' formats " %F{blue}%b%f %m%u%c %a "
zstyle ':vcs_info:*' enable git
zstyle ':vcs_info:*' check-for-changes true
zstyle ':vcs_info:*' stagedstr ' %F{green}✚%f'
zstyle ':vcs_info:*' unstagedstr ' %F{red}●%f'

precmd() {
    vcs_info
    print -P '%B%~%b ${vcs_info_msg_0_}'
}

PROMPT='%B%(!.#.$)%b '
```

Uso o esquema de cores Snazzy no iTerm, o que torna as cores do prompt ainda melhores. Minha configuração do ZSH tem mais algumas personalizações, como pesquisa de histórico, realce de sintaxe e autosugestões, mas vou deixar isso para outro post.

E é isso! Agora tenho um prompt ZSH personalizado que exibe o diretório de trabalho atual, branch e status do Git de maneira minimalista e colorida.
Código Completo do Prompt

```zsh
#!/usr/bin/env zsh

# Habilitar substituição de prompt
setopt prompt_subst

# Carregar módulo vcs_info
autoload -Uz vcs_info

# Configurar vcs_info para Git
zstyle ':vcs_info:*' enable git
zstyle ':vcs_info:*' check-for-changes true
zstyle ':vcs_info:git:*' formats ' %F{blue}%b%f %m%u%c %a'
zstyle ':vcs_info:*' stagedstr ' %F{green}✚%f'
zstyle ':vcs_info:*' unstagedstr ' %F{red}●%f'

# Função precmd para atualizar informações antes de cada prompt
precmd() {
    vcs_info
    # Exibir diretório atual em negrito e informações do Git
    print -P '%B%~%b ${vcs_info_msg_0_}'
}

# Prompt principal (símbolo # para root, $ para usuários normais)
PROMPT='%B%(!.#.$)%b '
```

Como Instalar

    Salve o código acima em ~/.zsh_custom_prompt.zsh

    Adicione ao seu arquivo .zshrc:

```zsh
source ~/.zsh_custom_prompt.zsh
```

Recarregue o ZSH:

```bash
source ~/.zshrc
```

Características do Prompt

    Múltiplas linhas: Diretório atual e informações do Git na primeira linha, símbolo do prompt na segunda

    Informações do Git:

        Mudanças staged (✚) em verde

Técnicas Avançadas (do Primeiro Artigo)
Adicionando Tempo de Execução

Para exibir o tempo de execução do último comando (útil para comandos longos):

```zsh
# Variável para armazenar o tempo inicial
typeset -g timer

preexec() {
    timer=$SECONDS
}

precmd() {
    vcs_info
    
    # Calcular e exibir tempo de execução se maior que 5 segundos
    if [[ -n $timer ]]; then
        local elapsed=$((SECONDS - timer))
        if [[ $elapsed -gt 5 ]]; then
            print -P "%F{yellow}${elapsed}s%f"
        fi
        unset timer
    fi
    
    print -P '%B%~%b ${vcs_info_msg_0_}'
}
```

Prompt do Lado Direito (RPROMPT)

Para informações menos críticas como a hora atual:

```zsh
# Adicionar ao precmd
RPROMPT='%F{242}%T%f'
```

Truncamento Inteligente de Caminhos

Para encurtar caminhos longos mantendo apenas os últimos diretórios:

```zsh
# No print do diretório, use:
print -P '%B%4~%b ${vcs_info_msg_0_}'
# Mostra no máximo 4 componentes do caminho
```

### Conclusão

Criar um prompt ZSH personalizado do zero foi uma experiência reveladora. Não apenas consegui um prompt mais rápido e adequado ao meu fluxo de trabalho, mas também ganhei uma compreensão mais profunda de como o ZSH funciona.

O processo me mostrou que você não precisa de frameworks complexos para ter um prompt funcional e bonito. Com um pouco de conhecimento das ferramentas básicas do ZSH, você pode criar algo perfeitamente adaptado às suas necessidades.

A chave é começar simples e adicionar funcionalidades gradualmente, testando cada mudança ao longo do caminho. E o melhor de tudo: agora entendo completamente como meu prompt funciona e posso modificá-lo facilmente conforme minhas necessidades evoluem.
